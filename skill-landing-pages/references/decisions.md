# Landing Page Decision Framework

Four dimensions determine what goes on your landing page and how much of it. Assess each before planning.

---

## Dimension 1: Product Price

**Free / Lead Magnet**
- Short page. Main benefit, product image, CTA. Mention "FREE" prominently.
- Minimal proof needed. People don't need much convincing for free stuff.

**Low to Medium Price**
- Build up perceived value. Cover secondary benefits.
- Add proof to support claims and reduce perceived risk.
- Show pricing.

**Expensive / Enterprise**
- Long page. Deep content.
- Multiple types of proof: text testimonials, video testimonials, reviews, client logos, usage stats, case studies.
- ROI calculator for B2B.
- Guarantees, financing options, support sections.

Rule: The more expensive the product, the longer the page.

---

## Dimension 2: Product Complexity

**Simple**
- Don't overcomplicate the presentation. Simplicity is a selling point.
- Brief benefits, quick CTA.

**Medium**
- Add a "How it works" section (1-2-3 pattern).
- Show the product in context.

**Complex**
- In-depth "How it works." Possibly video.
- Use cases showing different applications.
- Interactive demos if feasible.
- Feature breakdowns for technical audiences.

Rule: The more complex the product, the longer the page.

---

## Dimension 3: Consumer Awareness (Schwartz Scale)

**Unaware** (don't know they have a problem)
- Lead with the problem, not the product.
- Headline patterns: Pain Remover, Problem/Question, Question.
- Soft CTA: "Learn more."
- Thorough benefit explanation needed.
- Long education path before asking for commitment.

**Problem-Aware** (know the problem, don't know the solution)
- Headline patterns: Benefit, Easy Way, Pain Remover.
- Explain benefits first. Show how the product solves their problem.
- "How it works" section important.

**Solution-Aware** (know solutions exist, evaluating options)
- Headline patterns: Product Description, Product Category, Category & Audience.
- Balance benefits and features.
- Comparison sections useful. Differentiation matters.
- Proof from recognizable names.

**Product-Aware** (know your product, haven't bought)
- Headline patterns: Comparison, Double Benefit, Social Proof, Number One.
- Features and specifics matter more than broad benefits.
- Detailed pricing. Direct comparison to competitors.
- Testimonials from similar customers.

**Most Aware** (know your product, just need the push)
- Headline patterns: Promise, Testimonial, Social Proof.
- Strong CTA. Aggressive offer. Limited-time deals.
- Minimal education needed. Just remove final objections.
- Pricing, guarantees, bonuses.

Rule: The less informed the customer, the longer the page.

---

## Dimension 4: Market Sophistication

**New Market** (first-mover, novel category)
- Lead with new benefits. Nobody has seen a product like this before.
- Thorough benefit AND mechanism explanation (how it works).
- More education needed. Longer page.
- Create the category in the visitor's mind.

**Growing Market** (established category, competition emerging)
- Benefits still matter but start differentiating.
- Unique benefits over generic ones.
- Show why you're the better choice.

**Mature Market** (well-known category, many competitors)
- Focus on differentiation. Unique benefits, direct comparisons.
- Heavy proof to combat competitor brand value.
- If you're the established brand, leverage brand image.
- Features become more important (comparison shopping).

Rule: The newer the market, the longer the page.

---

## Putting It Together: Page Blueprint Process

1. Score each dimension (low/medium/high impact on page length)
2. Determine overall page depth needed
3. Select hero section recipe:
   - Low depth: Simple hero
   - Medium depth: Proof hero or Benefits hero
   - High depth: Benefits + Proof hero
4. Select headline pattern based on awareness level
5. Map below-the-fold sections:
   - Start with what the visitor wants to know NEXT
   - For complex products: benefits → how it works → features → proof → pricing
   - For simple products: benefits → proof → CTA
   - For expert audiences: features → comparison → pricing → proof
   - For unaware audiences: problem → benefits → how it works → proof → benefits again → CTA
6. Place proof adjacent to claims, not in a dedicated section
7. Place CTAs at every natural decision point
8. End with: Bottom CTA (headline + social proof + button) → Footer

---

## Quick-Reference: Section Order by Product Type

**SaaS**
Hero → How it works / Star feature → Main benefit → Feature list → Integrations → Testimonial → Support → Use cases → Pricing → Footer

**Physical Product**
Hero → Subheader/description → Features (collapsible) → Proof/results gallery → More features → Cross-selling → Specs → Guarantees & bonuses

**Mobile App (known)**
Hero with download CTA. That's it. Don't overexplain what people already know.

**Mobile App (new)**
Hero → Top benefits summary → Benefits in depth → Features grid → Press mentions → Use case testimonials → Logos + CTA → Footer

**Desktop App**
Hero (with self-selection options) → Benefits → Use cases → Proof → Awards + CTA

**Book / Info Product**
Hero (cover + headline + proof) → "What's it about?" (video + text) → "Why buy?" (reasons list) → "What's in it?" (table of contents/features) → Author bio/authority

## The Visitor's Inner Monologue

Use this to sequence your page. At each section, the visitor is asking one of these:

1. "What is this?" → Headline + hero image
2. "Why should I care?" → Main benefit
3. "How does it work?" → How it works section
4. "What exactly do I get?" → Features
5. "Can I trust these claims?" → Proof
6. "Will it work for me?" → Use cases / testimonials
7. "What do other people think?" → More proof
8. "How much does it cost?" → Pricing
9. "What if it doesn't work?" → FAQ / guarantees
10. "OK, how do I get started?" → Bottom CTA

Don't jump ahead. Don't answer question 8 before question 2. Follow the visitor's natural thought process.
